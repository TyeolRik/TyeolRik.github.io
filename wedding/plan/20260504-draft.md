# 모바일 청첩장 페이지 구현 계획

작성일: 2026-05-04

## 목적

`/root/TyeolRik.github.io/wedding` 디렉토리에 GitHub Pages에서 바로 서비스 가능한 모바일 청첩장 페이지를 만든다.

참고 대상은 `https://bojagicard.com/i/home.php?uid=kwon2982` 형식의 모바일 청첩장이며, 현재 덤프본은 `marry/` 디렉토리에 보관되어 있다. 실제 구현은 외부 서비스 의존성을 줄이고, 이 저장소의 정적 GitHub Pages 환경에서 동작하는 독립 HTML/CSS/JS 페이지로 작성한다.

## 기본 정보

- 신랑: 백승록
- 신부: 김현수
- 예식 일시: 2027년 05월 13일 목요일 오전 11시 30분
- 예식 장소: 식물관PH 수서
- 공개 URL 예상 경로: `https://tyeolrik.github.io/wedding/`
- 로컬 확인 경로: `http://127.0.0.1:4000/wedding/`

요청 원문에 `2027일 05월 13일`로 적혀 있으나, 구현에서는 `2027년 05월 13일`로 해석한다.

## Open Graph 적용 방침

기존 사이트 루트 `https://tyeolrik.github.io`와 청첩장 페이지 `https://tyeolrik.github.io/wedding/`는 서로 다른 HTML 문서이므로 Open Graph 메타 태그를 별도로 적용할 수 있다.

현재 저장소의 공통 head 파일 `_includes/head.html`도 페이지별 `og:title`, `og:type`, `og:description`, `og:url`을 일부 지원한다. 다만 `og:image`는 `page.image`를 `https://tyeolrik.github.io/img/{{ page.image }}`로 조합하는 구조라서 `wedding/opengraph/og.png`를 그대로 지정하기에는 맞지 않는다. 따라서 청첩장 페이지는 기존 사이트 레이아웃을 쓰지 않는 독립 HTML로 만들고, `wedding/index.html`의 `<head>`에 wedding 전용 Open Graph 메타를 직접 선언한다.

적용할 Open Graph 값:

- `og:title`: `백승록♡김현수 결혼`
- `og:type`: `website`
- `og:url`: `https://tyeolrik.github.io/wedding`
- `og:image`: `https://tyeolrik.github.io/wedding/opengraph/og.png`
- `og:description`: `백승록♡김현수의 결혼식에 당신을 초대합니다!`

추가 권장 메타:

- `title`: `백승록♡김현수 결혼`
- `description`: `백승록♡김현수의 결혼식에 당신을 초대합니다!`
- `twitter:card`: `summary_large_image`
- `twitter:title`: `백승록♡김현수 결혼`
- `twitter:description`: `백승록♡김현수의 결혼식에 당신을 초대합니다!`
- `twitter:image`: `https://tyeolrik.github.io/wedding/opengraph/og.png`

URL 표기는 사용자가 제공한 `https://tyeolrik.github.io/wedding`를 기준으로 하되, 실제 접속은 GitHub Pages에서 `/wedding/` trailing slash로 정규화될 수 있다. 공유 캐시 안정성을 위해 구현 시 canonical link는 `https://tyeolrik.github.io/wedding/`와 `https://tyeolrik.github.io/wedding` 중 하나로 통일한다.

## 디자인 방향

- 모바일 청첩장에 맞는 세로 스크롤 단일 페이지.
- 부드럽지만 과하게 장식적이지 않은 분위기.
- 사진이 주인공이 되도록 이미지 비율과 여백을 안정적으로 잡는다.
- 카드 중첩을 피하고, 섹션 단위의 넓은 배경과 제한된 본문 폭을 사용한다.
- 텍스트는 한국어 가독성을 우선해 적절한 행간과 작은 화면 대응을 적용한다.
- 외부 CDN 의존성 없이 시스템 폰트 또는 로컬 CSS만 사용한다.

## 기술 방침

- GitHub Pages/Jekyll에서 정적 파일로 동작하게 만든다.
- 별도 빌드 도구, npm 패키지, 외부 API는 사용하지 않는다.
- HTML/CSS/vanilla JS만 사용한다.
- 경로는 `/wedding/...` 공개 URL과 로컬 Jekyll 경로 모두에서 깨지지 않도록 상대 경로 중심으로 작성한다.
- Open Graph와 canonical처럼 외부 크롤러가 해석하는 URL은 절대 URL `https://tyeolrik.github.io/wedding/...`로 작성한다.
- 개인정보성 데이터인 계좌번호는 사용자가 제공한 그대로 노출한다.

## 접근성과 사용성

- 버튼에는 명확한 `aria-label` 또는 텍스트를 제공한다.
- 갤러리 모달은 `Escape` 키와 닫기 버튼으로 닫을 수 있게 한다.
- 이미지에는 의미 있는 `alt`를 넣는다.
- 계좌 복사 결과는 화면에 짧게 피드백한다.
- 터치 영역은 모바일에서 누르기 쉬운 크기로 잡는다.

## 검증 계획

1. `bundle exec jekyll serve` 실행.
2. `http://127.0.0.1:4000/wedding/` 접속 확인.
3. 모바일 폭 기준 확인:
   - 390px 내외 화면에서 텍스트 겹침 없음.

   - 계좌 복사 버튼이 동작함.
4. 데스크톱 폭 기준 확인:
   - 본문이 지나치게 넓어지지 않고 모바일 청첩장 형태로 중앙 정렬됨.
5. GitHub Pages 배포 경로 기준 확인:
   - `/wedding/gallery/photo*.jpg`
   - `/wedding/map/map.png`
   - `/wedding/opengraph/og.png`
   - `/wedding/styles.css`
   - `/wedding/script.js`
6. HTML head 확인:
   - `og:title`이 `백승록♡김현수 결혼`인지 확인.
   - `og:type`이 `website`인지 확인.
   - `og:url`이 `https://tyeolrik.github.io/wedding` 기준인지 확인.
   - `og:image`가 `https://tyeolrik.github.io/wedding/opengraph/og.png`인지 확인.
